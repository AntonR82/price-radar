# Price Radar — бэкенд (15 минут на запуск)

1. Создай приватный репозиторий на GitHub (например `price-radar`), загрузи в него ВСЁ содержимое этой папки (включая скрытую `.github/`).
2. Settings → Pages → Source: `main` branch → Save. Через минуту страница радара будет жить на `https://<логин>.github.io/price-radar/Price_Parity_Radar.html`.
3. Actions → включи workflows → запусти `price-radar-daily` вручную (Run workflow) — первый прогон соберёт `prices.json`.
4. Всё. Дальше сборщик ходит сам каждое утро в 08:20 по Дубаю; страница подхватывает свежие данные автоматически, история копится в `history/`.

## Что собирает
- **Курсы** — open.er-api.com (без ключа).
- **Бенчмарк** — World Bank Pink Sheet (месячные Urea/DAP/TSP/MOP FOB).
- **Розница** — BigHaat (Индия, самый надёжный: Shopify JSON), Daraz (Пакистан, Бангладеш), Jumia+Jiji (Кения), Jiji (Танзания, Уганда).

## Честные оговорки
- Маркетплейсы меняют разметку и ставят анти-ботов: адаптеры BigHaat/Jiji устойчивые (JSON-эндпоинты), Daraz/Jumia — парсинг, может потребовать подкрутки regex раз в квартал. Ошибка адаптера НЕ роняет сборку — просто грейд остаётся пустым, страница показывает последнее известное значение.
- Скрипт фильтрует упаковку по названию (кг из тайтла) — экзотические лоты дают выбросы, поэтому страница показывает медиану и n.
- Уважай robots: частота 1 раз/сутки, паузы 1,5 с между запросами — это read-only мониторинг публичных цен в личных целях.
